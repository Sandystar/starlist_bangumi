# CLI Run Flow

The current implementation direction is CLI-first. The WebView UI and SQLite
task history are not required for the core workflow. Each CLI invocation creates
one run folder containing all inputs, outputs, diagnostics, validation results,
and preview trees.

For a command-by-command interface reference, parameter list, artifact contract,
and safety notes, see [`docs/cli-reference.md`](cli-reference.md).

## Command

Analyze one first-level inbox folder:

```powershell
.\.venv\Scripts\python.exe tools\llm_debug_pipeline.py "[Folder Name]"
```

By default the folder is resolved under `media_library.source_path` from
`data/config.json`.

Useful options:

```powershell
--prompt "extra hint"
--tv-id 34742
--movie-id 382667
--output-root data/runs
```

If TMDB selection was wrong or incomplete, create a new remap run from an
existing run folder:

```powershell
.\.venv\Scripts\python.exe tools\remap_run.py "data/runs/{old_run}" --tv-id 35753
.\.venv\Scripts\python.exe tools\remap_run.py "data/runs/{old_run}" --movie-id 382667 --movie-id 123456
```

The remap command reuses the original folder name, source path, extra prompt,
identified work, and saved TMDB candidates. It rescans the source folder to
rebuild the full file list for subtitle binding, fetches TMDB details for the
manual IDs, reruns only `decide_mappings`, validates the result, and writes a
new run folder with `selection_mode: manual_remap`. The original run folder is
not modified.

When review is caused by missing TMDB episodes, provide a manual episode mapping
file:

```powershell
.\.venv\Scripts\python.exe tools\remap_run.py "data/runs/{old_run}" --mapping-file "data/runs/{old_run}/artifacts/manual_episode_mappings.json"
```

If `--mapping-file` is omitted, the command automatically uses
`artifacts/manual_episode_mappings.json` from the source run when it exists.
The file is intended to be generated by the future GUI:

```json
{
  "schema_version": "1.0",
  "tv_tmdb_id": "35753",
  "movie_tmdb_ids": [],
  "episode_mappings": [
    {
      "folder_path": "[VCB-Studio] Example Season",
      "file_name": "[VCB-Studio] Example [13(OVA)].mkv",
      "season_number": 0,
      "episode_number": 1,
      "reason": "Human review matched this OVA to TMDB S00E01."
    }
  ],
  "notes": ["Created from manual review UI."]
}
```

Manual episode mappings are applied as a patch on top of the previous
`mapping_output.expanded.json`: matching old decisions are replaced, matching
ignored files are removed, and all results still pass through the same strict
TMDB/source validation. The remap run also stores a copy of the mapping file at
`artifacts/manual_episode_mappings.json` so the new run remains auditable even
if the source run's review file changes later.

After reviewing a succeeded run, execute the organize phase:

```powershell
.\.venv\Scripts\python.exe tools\organize_run.py "data/runs/{timestamp}-{safe_folder_name}"
```

Destructive options are explicit:

```powershell
--delete-target-before
--overwrite-archive-target-before
--delete-source-after
--allow-failed-analysis
```

Resume a failed or interrupted organize attempt:

```powershell
.\.venv\Scripts\python.exe tools\organize_run.py "data/runs/{timestamp}-{safe_folder_name}" --resume-existing
```

List run records without opening each folder manually:

```powershell
.\.venv\Scripts\python.exe tools\list_runs.py
.\.venv\Scripts\python.exe tools\list_runs.py --latest-only
.\.venv\Scripts\python.exe tools\list_runs.py --status needs_review --source "Zero no Tsukaima"
.\.venv\Scripts\python.exe tools\list_runs.py --organize-status failed --json
```

Export a Markdown report:

```powershell
.\.venv\Scripts\python.exe tools\export_report.py "data/runs/{timestamp}-{safe_folder_name}"
.\.venv\Scripts\python.exe tools\export_report.py "{run_id}" --overwrite
```

Preview and execute run-folder cleanup:

```powershell
.\.venv\Scripts\python.exe tools\cleanup_runs.py --status failed
.\.venv\Scripts\python.exe tools\cleanup_runs.py --all --keep-latest-per-source 3
.\.venv\Scripts\python.exe tools\cleanup_runs.py --status failed --execute
```

## Run Folder Layout

Each run is saved as:

```text
data/runs/{timestamp}-{safe_folder_name}/
|-- manifest.json
|-- analysis_status.json
|-- organize_status.json
|-- organize_log.txt
|-- organize_log.jsonl
|-- timings.json
|-- requests/
|-- responses/
`-- artifacts/
```

Important files:

- `requests/*.json`: exact LLM request payloads.
- `requests/*.txt`: extracted readable prompt contents.
- `responses/*.json`: exact parsed LLM JSON responses.
- `artifacts/source_tree.txt`: scanned source tree.
- `artifacts/candidate_files.prompt.json`: grouped candidate list sent to mapping;
  includes videos plus subtitles that could not be matched to a video by filename.
- `artifacts/tmdb_*`: TMDB candidates and selected details.
- `artifacts/manual_remap_request.json`: manual TMDB IDs and source run path,
  present only in remap runs.
- `artifacts/manual_episode_mappings.json`: optional GUI-authored mapping file
  consumed by remap runs.
- `artifacts/mapping_output.expanded.json`: strict LLM mapping response after schema validation.
- `artifacts/work_plan.json`: program-validated plan.
- `artifacts/analysis_result.json`: status-ready analysis result.
- `artifacts/validation_summary.json`: compact status and counts.
- `artifacts/dry_run_result_tree.txt`: concise dry-run report.
- `artifacts/organized_target_tree.txt`: target-side organized tree preview.
- `organize_status.json`: latest organize status, options, target paths, and error details.
- `organize_log.txt`: human-readable organize log.
- `organize_log.jsonl`: structured organize log events.

## Run Index

`src/starlist_bangumi/run_index.py` is the read-only index over
`data/runs/`. It scans run folders and derives one `RunSummary` per folder from
these files:

- `manifest.json`
- `analysis_status.json`
- `organize_status.json`
- `artifacts/run_input.json`
- `artifacts/work_plan.json`
- `artifacts/analysis_result.json`
- `artifacts/manual_remap_request.json`

The index supports:

- listing most recent runs first;
- filtering by analysis status;
- filtering by organize status;
- filtering by source name/path substring;
- latest-only grouping by source path;
- loading a full `AnalysisResult` for a run.

SQLite is not part of the current MVP runtime. `data/runs/` is the source of
truth for history, diagnostics, analysis results, organize logs, and status.

## Markdown Reports

`tools/export_report.py` exports one run folder to `data/reports/` by default.
The filename is:

```text
{timestamp}_{safe_title}_v{analysis_version}.md
```

The command accepts a run id or a run folder path. Existing reports are not
overwritten unless `--overwrite` is passed.

Reports include:

- summary and status;
- source path and analysis version;
- selected TMDB entries;
- library and archive targets;
- validation counts;
- validated mappings;
- missing TMDB episodes and movies;
- rejected mappings;
- unmapped files;
- warnings;
- dry-run and organized target trees;
- organize status when available.

LLM request/response diagnostics are not embedded in the Markdown report. They
remain in the source run folder under `requests/` and `responses/`.

## Run Cleanup

`tools/cleanup_runs.py` previews or deletes run folders from `data/runs/`.
The command is dry-run by default and requires either `--all` or at least one
filter.

Supported filters:

- `--status succeeded|needs_review|failed|unknown`
- `--organize-status not_started|running|succeeded|failed`
- `--older-than-days N`
- `--keep-latest-per-source N`
- `--all`

Manual review runs that contain `artifacts/manual_episode_mappings.json` are
protected by default. Add `--include-manual` to allow deleting them.

Deletion happens only with `--execute`. Without `--execute`, the command prints
the selected folders and reasons without modifying the filesystem.

## Analyze Command Behavior

The analysis command performs no OpenList write operations. It:

- identifies the work from folder name and optional prompt;
- searches TMDB TV and Movie candidates;
- selects one TV series and zero or more movies;
- fetches every TMDB TV season with episodes;
- scans source files and builds a grouped candidate list of videos plus
  subtitles that cannot be filename-matched to a video;
- asks the LLM for strict grouped mapping decisions;
- validates mappings against TMDB and source candidates;
- binds same-name external subtitles after video validation;
- writes dry-run and target-side preview trees.

## Remap Command Behavior

The remap command performs no OpenList write operations. It:

- loads `artifacts/run_input.json`, `artifacts/identity.json`,
  `artifacts/extra_prompt.txt`, and saved TMDB candidates from the source run;
- requires `--tv-id`, one or more `--movie-id`, or both;
- can read `artifacts/manual_episode_mappings.json` to map specific files to
  TMDB TV season/episode numbers;
- creates a separate run folder under `data/runs/`;
- rescans the original source path for current video and subtitle files;
- fetches selected TMDB TV/Movie details;
- for TV, fetches seasons from TMDB TV details, not from CLI season hints;
- reruns `decide_mappings`, or applies the manual episode mapping patch when a
  mapping file is provided;
- writes the same analysis artifacts as a normal analysis run, plus
  `manual_remap_request.json` and `source_run_dir` in status files.

## Organize Command Behavior

The organize command performs OpenList write operations only after status
validation passes. It:

- reloads the run folder;
- recalculates analysis status from `WorkPlan` and TMDB artifacts;
- fails before OpenList copy/archive when the analysis is not `succeeded`;
- copies the full source folder into the archive target;
- copies only validated mappings into media-library targets;
- renames copied files/folders to planned target names;
- verifies archive target and every mapped media target;
- deletes the source only when `--delete-source-after` is set.

Default organize execution is intentionally strict: existing archive leaf
targets, existing mapped media files, and copied staging paths fail preflight
unless the matching explicit cleanup option is provided. This keeps accidental
duplicate copies visible.

`--resume-existing` is the recovery path for a previous failed or interrupted
organize attempt. In resume mode, the executor:

- skips a final archive leaf target that already exists;
- skips final mapped media files that already exist;
- renames existing copied staging media files to their planned target names;
- accepts a missing source folder only when archive and every mapped media
  target already verify successfully.

Resume mode cannot be combined with `--delete-target-before` or
`--overwrite-archive-target-before`. Use resume mode to continue a partial
execution, or use cleanup options without resume mode to intentionally recreate
targets.

## Status Contract

`analysis_status.json` and `artifacts/validation_summary.json` contain:

- `status`: `succeeded` or `needs_review`.
- `review_reason`: empty when succeeded.
- `validated`, `validated_videos`, `validated_subtitles`.
- `rejected`.
- `missing_tmdb_episodes`.
- `missing_movies`.
- `unmapped_files`.
- `library_targets`.
- `archive_target_path`.

The CLI analysis still performs no copy, move, archive, delete, or target
cleanup operations. It only writes local run artifacts.

The organize command performs OpenList operations. The archive target is
`archive_path + archive_path_template + source folder name`, so the full source
folder remains as a leaf directory under the canonical archive directory. The
executor copies validated mapped files into media-library targets, verifies the
archive leaf and mapped files, and deletes the source only when
`--delete-source-after` is explicitly set.

`organize_status.json.options.resume_existing` records whether the run used the
resume path, so the run folder remains auditable after recovery.

`tools/organize_run.py` recalculates analysis status from the saved `WorkPlan`
before running. A stale `analysis_result.json` that says `succeeded` can still be
blocked if the current validation rules find missing TMDB episodes or rejected
mappings.

## Validation Rules

The program validates the LLM mapping output after `decide_mappings`:

- grouped `folder_path + file_name` decisions must resolve to candidate files,
  and primary media decisions must use video files.
- TV mappings must match a concrete TMDB season and episode.
- Movie mappings must match a selected TMDB movie ID.
- Duplicate source mappings and duplicate target paths are rejected.
- External subtitles are added only after their video mapping is validated.
- Missing episodes are computed from the fetched TMDB episode list, not from the
  LLM coverage scope. Every TMDB season/episode must have a mapped source file.
- Files that do not enter the media library remain in `unmapped_files`; extra
  source videos do not require review by themselves.

The target-side preview tree is generated from validated mappings only.
